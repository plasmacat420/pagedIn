# PagedIn

> **Paste your resume. Get a live website. Free, forever.**

PagedIn turns any resume (plain text or PDF) into a beautiful GitHub Pages site in under 60 seconds — no GitHub account required.

---

## How it works

1. Paste resume text or upload a PDF
2. Claude AI parses and structures the data
3. Preview the generated site (two themes available)
4. One-click deploy to GitHub Pages — either via the PagedIn hosting account or your own GitHub

## Local development

### Prerequisites

- Python 3.11+
- Node.js 20+
- An Anthropic API key
- A GitHub Personal Access Token (for the owner deployment account)

### Backend

```bash
cd backend
python -m venv .venv
source .venv/bin/activate      # Windows: .venv\Scripts\activate
pip install -r requirements.txt

# Copy and fill in env vars
cp .env.example .env

uvicorn main:app --reload --port 8000
```

Visit http://localhost:8000/docs for the interactive API docs.

### Frontend

```bash
cd frontend
npm install

# Copy and fill in env vars
cp .env.example .env

npm run dev
```

Visit http://localhost:5173

## Deployment

### Backend → Render

1. Push this repo to GitHub
2. Create a new **Web Service** on [Render](https://render.com), pointing to the `backend/` directory
3. Render will detect `render.yaml` automatically
4. Set the following environment variables in the Render dashboard:
   - `ANTHROPIC_API_KEY`
   - `GITHUB_OWNER_TOKEN` — PAT for the owner deployment account (needs `repo` scope)
   - `GITHUB_OWNER_ORG` — GitHub org or username where PagedIn sites are deployed
   - `GITHUB_CLIENT_ID` — GitHub OAuth App client ID
   - `GITHUB_CLIENT_SECRET` — GitHub OAuth App client secret
   - `FRONTEND_URL` — your GitHub Pages URL (for CORS)
5. `HMAC_SECRET` is auto-generated by Render

### Frontend → GitHub Pages

1. In your GitHub repo settings → Secrets and variables → Actions, add:
   - `VITE_BACKEND_URL` — your Render backend URL
   - `VITE_GITHUB_CLIENT_ID` — GitHub OAuth App client ID
   - `VITE_BASE_PATH` — e.g. `/pagedin/` if your repo is at `github.com/you/pagedin`
2. Enable GitHub Pages in repo settings → Pages → Source: **GitHub Actions**
3. Push to `main` — the workflow in `.github/workflows/deploy-frontend.yml` will build and deploy automatically

### GitHub OAuth App setup (for self-deploy)

1. Go to [GitHub Developer Settings → OAuth Apps](https://github.com/settings/developers) → New OAuth App
2. Set **Authorization callback URL** to: `https://your-pagedin-site.github.io/oauth/github`
3. Copy the Client ID and Client Secret into your env vars

### Owner deployment GitHub account

1. Create a dedicated GitHub account or org (e.g. `pagedin-sites`)
2. Generate a **Classic Personal Access Token** with `repo` scope
3. Set as `GITHUB_OWNER_TOKEN` and `GITHUB_OWNER_ORG`

## Rate limits

| Endpoint           | Limit              |
|--------------------|--------------------|
| POST /parse        | 3 requests / hour  |
| POST /generate     | 3 requests / hour  |
| POST /deploy/pagedin | 2 deploys / day  |
| POST /deploy/self  | 3 deploys / day    |

Uses a sliding window algorithm. Backed by Redis in production, in-memory in dev. Set `REDIS_URL` for production.

## Environment variables reference

### Backend (`backend/.env`)

| Variable | Required | Description |
|---|---|---|
| `ANTHROPIC_API_KEY` | Yes | Claude API key for resume parsing |
| `GITHUB_OWNER_TOKEN` | Yes | GitHub PAT for owner deployment account |
| `GITHUB_OWNER_ORG` | Yes | GitHub org/user for hosted sites |
| `GITHUB_CLIENT_ID` | Yes | OAuth App client ID |
| `GITHUB_CLIENT_SECRET` | Yes | OAuth App client secret |
| `HMAC_SECRET` | Yes | Secret for signing deploy tokens |
| `FRONTEND_URL` | Yes | Frontend URL for CORS |
| `REDIS_URL` | No | Redis for production rate limiting |

### Frontend (`frontend/.env`)

| Variable | Required | Description |
|---|---|---|
| `VITE_BACKEND_URL` | Yes | Backend API URL |
| `VITE_GITHUB_CLIENT_ID` | Yes | GitHub OAuth App client ID |
| `VITE_BASE_PATH` | No | Base path for GitHub Pages subdirectory |

## Security

- IP-based sliding window rate limiting on all POST endpoints
- HMAC-signed deploy tokens prevent arbitrary HTML being deployed to the owner GitHub account
- Honeypot fields catch automated bots (silent fake-success response)
- 50 KB request size limit on parse/generate
- User-agent filtering rejects known bot patterns
- All abuse attempts logged server-side
- CORS restricted to frontend origin only
- GitHub OAuth tokens stored in frontend memory only (not localStorage)
